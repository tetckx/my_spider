# ITF 项目数据爬虫 - 用户使用手册

## ⚡ 快速开始

### 1. 安装依赖

```bash
cd ITF
npm install
```

### 2. 修改配置

编辑 `config.json` 文件，设置要查询的参数：

```json
{
  "keywords": "能源",
  "lang": "sc",
  "pages": 50
}
```

### 3. 运行爬虫

```bash
node scrape/scrape.js
```

---

## 📖 config.json 配置说明

| 参数 | 说明 | 可选值 | 示例 |
|------|------|--------|------|
| keywords | 搜索关键词 | 任意中文/英文 | `能源`, `电池`, `太阳能` |
| lang | 网站语言 | `sc` (简体中文), `en` (英文) | `sc` |
| pages | 目标页数 (每页10条) | 数字 | `50` |

---

## 💡 使用示例

### 示例1: 默认运行 (能源, 简体中文)

```bash
node scrape/scrape.js
```

### 示例2: 命令行参数覆盖

```bash
# 覆盖config.json设置
node scrape/scrape.js sc 电池 30
```

### 示例3: 查询英文网站

```bash
node scrape/scrape.js en battery 20
```

---

## 📁 输出文件

运行完成后，会在项目根目录生成三个文件：

| 文件名 | 说明 |
|--------|------|
| `itf-{关键词}-projects.csv` | CSV格式 |
| `itf-{关键词}-projects.json` | JSON格式 |
| `itf-{关键词}-projects.xlsx` | Excel格式 |

---

| 字段名 | 说明 | 示例 |

## ⚠️ 注意事项

1. **浏览器窗口**: 默认会弹出浏览器窗口 (非无头模式)，不会干扰你操作
2. **边爬边存**: 数据会实时保存到文件，无需等待全部完成
3. **网络要求**: 爬取过程中需要保持网络连接

---

## 🔧 故障排除

### 问题: 数据为空

- 检查关键词是否正确
- 检查网络是否正常

### 问题: 运行报错

- 确保已安装依赖: `npm install`
